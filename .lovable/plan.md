# خطة التحديثات

## 1) إدارة الصلاحيات (للمدير فقط)

في صفحة **الإعدادات** → تبويب جديد **"الصلاحيات"**:
- جدول: الصفوف = الصلاحيات (مجمّعة حسب الموديول)، الأعمدة = الأدوار (محاسب، مدير مالي، مستخدم).
- خانات اختيار (✓) لكل تقاطع — التحديث فوري على جدول `role_permissions`.
- دور **admin** يبقى ثابتاً (كل الصلاحيات، غير قابل للتعديل).

**قاعدة البيانات:** لا تغيير في الجداول — فقط واجهة CRUD على `role_permissions` (موجودة بالفعل + سياسة `users.manage`).

## 2) دمج الممول مع المشروع

**النموذج الجديد:**
- إضافة عمود `project_code` (نص، اختياري) و `is_project` (boolean) إلى جدول `funders`.
- نموذج "ممول جديد" يستبدل حقل **الهاتف** بـ **رقم المشروع**.
- خانة اختيار "هذا الممول مشروع أيضاً" (افتراضياً مفعّلة).
- إذا فُعّلت → عند الحفظ، يُنشأ سجل في `projects` مرتبط بنفس الاسم/الكود تلقائياً (عبر تريغر).
- إذا لم تُفعّل → يبقى ممولاً فقط (لا يظهر في قوائم المشاريع).

**التنظيف:**
- ربط الممولين الحاليين بالمشاريع المتطابقة بالاسم.
- إخفاء حقل `phone` من نموذج الممول (يبقى في DB للتوافق مع البيانات القديمة).

**صفحات المشاريع:** تبقى منفصلة لكن المشاريع المُنشأة من ممول تكون للقراءة فقط (تُحرَّر من صفحة الممول).

## 3) التتبع بالرقم في التقارير

في صفحة **التقارير** → شريط بحث علوي:
- حقل واحد: "ابحث برقم صك أو رقم مشروع"
- عند الإدخال يعرض بطاقة تفاصيل:
  - **للصك:** المبلغ، المنصرف، المتبقي، الممول، قائمة المصروفات المرتبطة (مع التواريخ والمشاريع والفئات).
  - **للمشروع:** إجمالي المصروفات، الصكوك المموِّلة، تقسيم حسب الفئة، آخر الحركات.

## 4) مرفق Excel للمصروفات

- نموذج المصروف **يدعم بالفعل** مرفقاً عاماً (`attachment_url` + bucket `expense-attachments`).
- توسيع: السماح بمرفق إضافي من نوع Excel (`.xlsx`, `.xls`, `.csv`) في حقل ثانٍ `excel_attachment_url` + رابط تنزيل من جدول المصروفات.

## التقنية باختصار

**Migrations:**
- `ALTER TABLE funders ADD project_code TEXT, ADD is_project BOOLEAN DEFAULT true`
- `ALTER TABLE expenses ADD excel_attachment_url TEXT`
- تريغر `funders_sync_project()`: عند insert/update لممول بـ `is_project=true` يُنشئ/يحدّث سجل `projects` بنفس الاسم والكود.
- backfill: ربط الممولين الحاليين بالمشاريع المتطابقة.

**Frontend:**
- `settings.tsx`: تبويب "الصلاحيات" مع مصفوفة checkboxes.
- `funders.index.tsx` + `funders.$funderId.tsx`: استبدال phone بـ project_code + خانة is_project.
- `reports.tsx`: شريط بحث مع `ReferenceTracker` component.
- `expenses.tsx`: حقل رفع Excel ثانٍ + عرض رابطه في الجدول.

## خارج النطاق
- لا تغيير على القيود اليومية، فئات المصروفات، أو الحسابات النقدية.
- لا حذف لجدول `projects` — يبقى مع علاقة 1-1 ضمنية مع الممول عند `is_project=true`.
